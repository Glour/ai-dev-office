# Security Review Workflow

Поток:

1. Security Officer фиксирует scope: продукт, интеграция, релиз, рекламный инструмент или задача.
2. Проверяет секреты, токены, доступы, зависимости, обработку данных, логи и внешние интеграции.
3. Формирует findings с severity.
4. Дает mitigation plan.
5. Возвращает go/no-go decision владельцу.

Запрет: не печатать секреты и не одобрять запуск при неизвестных доступах или неясном сборе данных.
